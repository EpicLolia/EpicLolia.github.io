【More Effective C++】2 操作符 Operators

发表于 2022-10-25 分类于 computer-science ， programming-language ， cpp 本文字数： 1.5k 阅读时长 ≈ 6 分钟

条款 5 对定制的“类型转换函数”保持警觉

隐式类型转换函数的两种定义：

  • 能够以单一自变量构造的构造函数（构造函数只有一个参数 or 除第一个参数外都有默认值）
  • operator type_name() { }

缺点：导致非预期的函数被成功调用。（理应报错的函数却被“蹩脚”的隐式转换成功避过）

合理的做法：

  • 使用 explicit 关键字防止单变量构造函数被用作隐式转换（还有一个做法就是使用 proxy class(代理类)）

  • 明确定义一个函数来进行转换（例如 string 的 c_str() 函数）

编译器的隐式转换只允许转换一次，即不会出现编译器隐式 A->B->C 两次转换

请记住

  • 允许编译器执行隐式转换害处将多过好处。所以不要提供转换函数，除非你确定你需要他们。
