Hexo 中的常用特性

发表于 2026-05-05 更新于 2026-05-17 分类于 misc 阅读次数： 本文字数： 1k 阅读时长 ≈ 4 分钟

本文记录了搭建本博客过程中用到的 Hexo 配置与自定义脚本，包括打包发布、主题设置、资产路径处理等常用操作，方便日后查阅和复用。

Hexo 打包/发布流程

使用 Next 主题

参考hexo-theme-next 文档

  1. 执行 npm install hexo-theme-next --save 安装 Next 主题，然后在 _config.yml 中设置 theme: next。
  2. 执行 cp node_modules/hexo-theme-next/_config.yml _config.next.yml 导入 Next 主题的配置文件。

自定义打包流程

Hexo 打包时会自动加载 <ROOT>/scripts/ 目录下的所有 .js 文件，开发者可以根据需要在 scripts/ 目录下添加自定义脚本。

打包时自动生成 categories 字段

本博客仓库主要通过文件目录结构来维护文章的分类信息，为减少维护成本，本仓库通过脚本在打包时根据文章在 _posts/ 下的目录结构自动生成 categories 字段。

例如文章路径为 _posts/game-dev/graphics/games202-notes.md，打包时会自动设置 categories: ["game-dev", "graphics"]。

因此只需合理组织文章所在的目录层级，categories 就会相应生成，无需在文章的 Front Matter 中维护 categories 字段。

打包时转换资产路径

本仓库的资产引用约定为：在文章 article.md 中通过同名目录引用资产，如 ![图片](article/pic.png)。这样在本地编辑器中路径可以正确解析，方便预览。

但 Hexo 的资产文件夹（Post Asset Folder）机制要求使用不带目录前缀的裸文件名（如 pic.png）来引用同名资产文件夹中的资源。需要先在 _config.yml 中开启相关选项：

再通过脚本在打包时自动去除资产目录前缀。例如在 _posts/game-dev/graphics/games202-notes.md 中写 ![图片](games202-notes/pic.png)，打包时自动转换为 ![图片](pic.png)，再由 Hexo 解析为正确的站点路径。

只有以文章 slug 同名目录为前缀的路径（如 article/pic.png 或 ./article/pic.png）会被转换，外部链接和其他路径不受影响。

支持 LaTex 格式

next 主题自带 LaTex 格式的适配。我们只需要在_config.next.yml中开启mathjax

并在需要使用 LaTex 的文章的头部的 yaml 标签中加上mathjax: true即可。

浏览量统计

Next 主题支持使用不蒜子进行浏览量统计，在_config.next.yml中配置即可。
